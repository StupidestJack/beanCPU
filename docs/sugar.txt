=== 豆子架構語法糖與編譯器指令手冊 (sugar.txt) ===

本文件記錄編譯器層級支援的擴充語法與資料定義指令。

1. 前處理指令: DEFINE [名稱] [數值/暫存器]
   ＊ 功能：在編譯階段進行全域文字替換。
   ＊ 範例：DEFINE SCREEN 0xFF00

2. 暫存器擴充語法: MOV Rn, Rm
   ＊ 硬體實作：SUB Rn, Rn (0x03) + ADD Rn, Rm (0x02)
   ＊ 功能描述：將暫存器 Rm 的值複製到 Rn。
   ＊ 備註：此語法糖不佔用堆疊，但會產生兩條硬體指令，消耗 2 個指令空間。

3. 資料定義指令: .FILL [數值 或 標籤名稱]
   ＊ 功能：在當前位址寫入一個 16 位元數值。
   ＊ 標籤引用：若參數為標籤，則寫入該標籤所在的記憶體位址。
   ＊ 空間佔用：每個 .FILL 佔用記憶體中的 1 個字 (2 bytes)。

4. 檔案引用指令: USING [檔案路徑]
   ＊ 功能：引用外部檔案內容。檔案不存在會報錯，並且會放到主文件（輸入參數時的那一個文件）的末尾，而非直接替換，避免順序執行而出錯
   ＊ 備註：USING得到的檔案文件目前是使用組譯器的相對路徑，而非檔案的相對路徑，這一點需要注意（之後可能會改，但先這樣）


注意事項：
＊ 執行 MOV Rn, Rm 雖然目前實作為 SUB+ADD，但若未來改回 PUSH/POP 實作，仍需注意堆疊指標 SP 是否有效。
＊ 建議將 .FILL 定義的資料區放在 BRK (0x00) 之後，避免 CPU 將純資料誤判為指令執行。
＊ 除上述擴充語法外，其餘指令必須嚴格遵循 dirc.txt 的硬體 Opcode 定義。
